# TOTAL 가이드-화면 차이/누락 분석 (전수)

작성일: 2026-03-23  
대상 범위: `전체 공기산정(TOTAL)`  
제외 범위: `로그인`, `사용자 가이드`

## 0) 결론

- 현재 가이드는 TOTAL의 주요 흐름은 설명하지만, **버튼/단축키/세부 동작 기준으로는 전수 커버가 아니다**.
- 특히 누락이 큰 구간은 아래다.
  - `공기산정기준(/schedule-master)`의 테이블/간트 고급 기능
  - `가동률`의 통합/혼합/분리 및 process 상속·분리 제어
  - `표준품셈`의 보기 모드, 계층 편집, 자동저장 동작
  - `생산성 근거 3종`의 결과표/기준표/산출내역 상세 조작

## 1) 분석 기준

- 가이드 기준: `frontend/src/assets/공기검토플랫폼_사용자_가이드.md` (TOTAL 섹션)
- 화면 기준(코드):
  - 라우팅: `frontend/src/App.jsx`
  - 공통 헤더/메뉴: `frontend/src/components/layout/Header.jsx`
  - 홈: `frontend/src/pages/Home.jsx`
  - 가동률: `frontend/src/pages/OperatingRate.jsx`
  - 표준품셈: `frontend/src/pages/TotalCalc.jsx`, `frontend/src/components/cpe/DataTable.jsx`
  - 생산성 근거: `frontend/src/pages/CIPBasisList.jsx`, `frontend/src/pages/PileBasisList.jsx`, `frontend/src/pages/BoredPileBasisList.jsx`, `frontend/src/components/cpe/EditableTable.jsx`
  - 공기산정기준(테이블/간트):
    - `frontend/src/pages/ScheduleMasterList.jsx`
    - `frontend/src/components/cpe/schedule/*`
    - `frontend/src/components/cpe/GanttChart.jsx`
    - `frontend/src/components/cpe/GanttChartArea.jsx`
    - `frontend/src/components/cpe/gantt/ui/*`
  - 요약장표: `frontend/src/pages/Summary.jsx`

- 상태 표기:
  - `기재`: 가이드에 명시적으로 존재
  - `부분기재`: 가이드는 있으나 버튼/세부 동작까지는 미기재
  - `누락`: 가이드에 없음
  - `불일치`: 가이드와 화면 표현/동작이 다름

## 4) 불일치/누락 우선순위 (가이드 보강 기준)

### P1 (사용 플로우 직접 영향)

- `SCH-T-01~18`, `SCH-C-01~15`, `SCH-G-01~20` (공기산정기준 핵심 조작)
- `OPR-03~07` (가동률 핵심 모드/상속 제어)
- `STD-01~08` (표준품셈 실사용 편집 흐름)

### P2 (운영 정확도/재현성 영향)

- 단축키 전체(섹션 3)
- AI Compare/적용 상세(`AI-04`, `AI-05`)
- 생산성 근거 3종 상세 선택 로직(CIP/PILE/BORED의 타입/공법/규격 선택)

### P3 (표현/용어 정합)

- `공사기간 산정 기준` vs 실제 메뉴명 `공기산정기준`
- 드래그 제약(같은 대공종 내 이동) 명시
- 링크 편집 상세(FS/SS/FF/SF, Lag) 명시

## 5) 정리

- 이 문서는 TOTAL 화면 코드 기준으로 **버튼/키/세부 동작을 전수 수집**해 가이드 반영 상태를 매긴 결과다.
- 현재 가이드는 흐름 설명용으로는 유효하지만, 운영 매뉴얼 기준(세부 기능/버튼/단축키)으로는 누락이 크다.
- 다음 업데이트 시 우선순위는 `공기산정기준 -> 가동률 -> 표준품셈 -> 근거3종` 순이 적절하다.
