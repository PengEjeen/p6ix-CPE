## 3) 상세 결과

### [HIGH-01] 운영 보안 설정이 개발값으로 고정
- 근거:
  - `backend/backend/settings.py:36` `DEBUG = True`
  - `backend/backend/settings.py:38` `ALLOWED_HOSTS = ["*"]`
- 영향:
  - 디버그 정보 노출 가능성 증가, 호스트 헤더 기반 공격면 확대
- 권고:
  1. `DEBUG = env.bool("DEBUG", default=False)`로 전환
  2. `ALLOWED_HOSTS`를 운영 도메인 화이트리스트로 제한
  3. 배포 파이프라인에서 `DEBUG=False` 강제 검증

### [HIGH-02] CORS 과개방 + Credentials 허용 조합
- 근거:
  - `backend/backend/settings.py:76` `CORS_ALLOW_ALL_ORIGINS = True`
  - `backend/backend/settings.py:77` `CORS_ALLOW_CREDENTIALS = True`
- 영향:
  - 세션 인증 모드 사용 시, 임의 오리진에서 인증 컨텍스트를 가진 요청/응답 접근 위험 증가
- 권고:
  1. `CORS_ALLOWED_ORIGINS` 명시 화이트리스트 사용
  2. 운영에서 `CORS_ALLOW_ALL_ORIGINS=False`
  3. 세션 모드 사용 시 `CSRF_TRUSTED_ORIGINS`도 최소화

### [HIGH-03] HTTPS 강제/보안 쿠키/HSTS 미적용
- 근거:
  - `backend/backend/settings.py:209` `SESSION_COOKIE_SECURE` 기본 `False`
  - `backend/backend/settings.py:211` `CSRF_COOKIE_SECURE` 기본 `False`
  - `check --deploy` 경고: `security.W004`, `W008`, `W012`, `W016`
- 영향:
  - 중간자 공격 환경에서 세션/CSRF 토큰 탈취 리스크 증가
- 권고:
  1. `SECURE_SSL_REDIRECT=True`
  2. `SESSION_COOKIE_SECURE=True`, `CSRF_COOKIE_SECURE=True`
  3. `SECURE_HSTS_SECONDS`(+ includeSubDomains/preload 정책 검토) 적용

### [HIGH-04] 서버 내부 에러 상세가 API 응답으로 노출
- 근거:
  - `backend/cpe_all_module/views/construction_schedule.py:181-182, 246-247`
  - `backend/user/views.py:95-101`
  - `backend/user/google_auth.py:93, 98`
  - `backend/cpe_module/utils/gemini_runner.py:175-176, 202-203` (상위 API로 전달)
- 영향:
  - 내부 구성/라이브러리/예외 원인 노출로 공격자 정찰에 유리
- 권고:
  1. 외부 응답은 일반화된 에러 코드/메시지로 통일
  2. 상세 예외는 서버 로깅으로만 보관
  3. `print/traceback.print_exc()` 제거 후 구조화 로깅 적용

### [MEDIUM-01] JWT 토큰의 LocalStorage 저장
- 근거:
  - `frontend/src/api/axios.js:60, 93, 103-105`
- 영향:
  - XSS 발생 시 토큰 탈취 가능성 증가
- 권고:
  1. 가능하면 HttpOnly Secure Cookie 기반으로 전환
  2. 최소한 CSP 강화 + 입력/출력 인코딩 + 서드파티 스크립트 통제

### [MEDIUM-02] 인증 엔드포인트 요청 제한(Throttle) 부재
- 근거:
  - `backend/backend/settings.py`의 `REST_FRAMEWORK`에 `DEFAULT_THROTTLE_CLASSES/RATES` 설정 없음
  - 공개 인증 엔드포인트 존재: `backend/user/urls.py` (`register`, `login`, `google/login`, `keycloak/login`)
- 영향:
  - 계정 탈취 시도/자원 소모형 공격에 취약
- 권고:
  1. DRF throttle 클래스/레이트 설정 추가(anon/user 별도)
  2. 로그인/토큰 발급 엔드포인트 별도 강화

### [LOW-01] 공개 회원가입 엔드포인트 운영정책 불일치 가능성
- 근거:
  - `backend/user/views.py:17-21` (`RegisterView`, `AllowAny`)
  - `backend/backend/settings.py:200` `LEGACY_LOCAL_LOGIN_ENABLED` 기본 `False`
- 영향:
  - 로컬 로그인 비활성 정책 환경에서 불필요 계정 생성 가능
- 권고:
  1. 로컬 로그인 비활성 시 회원가입도 비활성화 정책 연동
  2. 필요 시 승인 워크플로우/도메인 제한 추가
