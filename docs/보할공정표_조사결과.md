# 보할공정표(補割工程表) 조사 결과

> 작성일: 2026-04-14  
> 목적: P6IX 공기산정 플랫폼 보할공정표 기능 구현을 위한 사전 조사

---

## 1. 정의

**보할(補割)** = 보조할당(補助割當)의 준말

> 전체 공사를 100%로 봤을 때, 각 공종이 차지하는 **금액 비중(%)** 을 할당하는 것

보할공정표는 **공종별 공사비 배분 + 시간 배분**을 결합한 문서로,  
발주처 자금투입 계획 및 기성(중간지불) 관리의 기초 자료로 사용된다.

---

## 2. 보할비율 결정 방법

### 기준: 공사비(금액)

```
보할비율(%) = 각 공종 공사비 ÷ 전체 공사비 × 100
```

- **표준**: 공사비 기준 (내역서의 금액 기반)
- Man-hours / 공기(일수) 기준도 사용 가능하나 비표준

### 예시

| 공종 | 공사비 | 보할비율 |
|------|--------|---------|
| 가설공사 | 50억 | 5% |
| 토공사 | 100억 | 10% |
| RC공사 | 350억 | 35% |
| 마감공사 | 300억 | 30% |
| 기타 | 200억 | 20% |
| **합계** | **1,000억** | **100%** |

---

## 3. 실제 양식 구조

### 행(Row) 구조

```
대공종
  └─ 중공종
       └─ 소공종 (세부공종) ← 실제 입력 단위
```

가장 깊은 계층인 **소공종(세부공종)** 단위로 월별 배분을 입력하고,  
상위 계층은 SUM으로 자동 집계된다.

### 열(Column) 구조

| 공종코드 | 공종명 | 공사비(원) | 1월 | 2월 | 3월 | … | 합계(원) | 비율(%) | 누적(%) |
|---------|--------|---------|-----|-----|-----|---|---------|--------|--------|
| 01 | 가설공사 | 50억 | 25억 | 25억 | — | | 50억 | 5% | 5% |
| 02 | 토공사 | 100억 | — | 50억 | 50억 | | 100억 | 10% | 15% |
| … | | | | | | | | | |
| **합계** | | **1,000억** | | | | | | **100%** | |

- 월 단위 또는 **주(Week) 단위** (1개월 = 4주)로 세분화 가능
- 우측: 합계(원), 비율(%), 누적(%) 컬럼

---

## 4. 월별 배분 방식

### 방식 1: 균등 배분 (가장 일반적)

공종의 작업 기간 내에 금액을 균등하게 분산

```
월 배분액 = 공사비 × (해당 월의 작업일수 / 전체 작업일수)
```

### 방식 2: 강도 차등 배분

피크 시기에 더 많은 금액 배분 (엑셀에서 1, 2, 3 입력)

```
월 배분액 = 공사비 × (해당 월 강도값 / 전체 강도값 합계)

예) 기초공사 100억, 3개월 진행 (강도: 1, 2, 1)
  - 1월 = 100억 × 1/4 = 25억
  - 2월 = 100억 × 2/4 = 50억
  - 3월 = 100억 × 1/4 = 25억
```

### 방식 3: 수동 입력

실무 경험 기반으로 각 셀에 직접 금액 또는 비율 입력

---

## 5. 계층 집계 방식

Bottom-up (하향식 → 상향식) SUM 집계

```
소공종 월별 금액
  → SUM → 중공종 월별 금액
      → SUM → 대공종 월별 금액
          → SUM → 전체 월별 합계
              → 누적 공정률(%) → S-curve
```

---

## 6. 보할공정표 / S-curve / 기성공정표 관계

```
[기본 공정표 (Gantt/CPM)]
  - 각 공종의 작업 순서, 기간
        ↓ (금액 할당)
[보할공정표] ← 계획(PLAN)
  - 공종별 금액 × 기간 배분
  - 월별 투입 계획액
  - 누적 계획 공정률(%)
        ↓
[S-curve — 계획선]
        ↑ 비교
[기성공정표] ← 실적(ACTUAL)
  - 매월 실제 완성 공종/금액 기록
  - 누적 실적 공정률(%)
        ↓
[S-curve — 실적선]
        ↓
[편차 분석]
  편차 = 실적 누적% - 계획 누적%
  음수 → 지연, 양수 → 선행
```

### S-curve 형태

```
100% ┤                              ●
     │                          ●
 75% ┤                      ●
     │                  ●
 50% ┤             ●
     │         ●
 25% ┤     ●
     │  ●
  0% ┼──────────────────────────────
     1월  3월  5월  7월  9월  11월
          ─── 계획   ─── 실적
```

---

## 7. 기성(중간지불) 계산 예시

```
기성금액 = 전체 공사비 × 누적 실적 공정률(%)

예)
전체 공사비: 1,000억
1월 실적:
  - 가설공사 100% 완료 (50억)
  - 토공사 20% 완료 (100억 × 20% = 20억)
  → 1월 기성청구액 = 70억
  → 1월 실적 공정률 = 70 / 1,000 = 7%
```

## 8. 발주처 제출 형식

- 착공 후 **60일 이내** 제출 (관급공사 기준)
- 포함 항목:
  - 보할공정표 (대/중/소공종별 월별 배분표)
  - S-curve 그래프
  - 기본 공정표 (Gantt 또는 CPM)
  - 내역서 요약

## 9. 현재 P6IX 시스템과의 갭 분석

### 현재 있는 것

| 필드 | 용도 |
|------|------|
| `main_category` | 대공종 |
| `process` | 중공종 |
| `sub_process` | 소공종 |
| `work_type` | 세부 작업명 (최하위) |
| `calendar_days` | 작업 기간 (공기) |
| `startDay` | 작업 시작일 (일 기준) |
| `working_days` | 실제 작업일수 |

### 없는 것 (추가 필요)

| 필드 | 필요 이유 |
|------|----------|
| `cost` (공사비) | 보할비율 계산의 표준 기준 |
| `weight` (보할비율%) | 직접 입력 또는 cost로 자동계산 |
| `monthly_progress` (월별 실적) | 기성 관리용 |

### 구현 선택지

| 방식 | 설명 | 장단점 |
|------|------|--------|
| **A: 공사비 기준** | `cost` 필드 추가 → 자동 보할비율 계산 | 표준 방식, 필드 추가 필요 |
| **B: 공기 기준 대체** | `calendar_days` 비율로 보할 근사 | 빠른 구현, 정확도 낮음 |
| **C: 수동 입력** | 사용자가 `weight(%)` 직접 입력 | 가장 유연, 자동화 없음 |

## 10. 구현 범위 제안 (단계별)

### 1단계 — 보할 설정 (MVP)
- 세부공종별 `weight(%)` 입력 (합계 100% 검증)
- 자동배분 보조: `calendar_days` 비율로 초기값 제안
- 전체 진척률 = Σ(weight × progress / 100)

### 2단계 — 월별 계획 배분
- `startDay` + `calendar_days` → 월별 자동 배분
- 세부공종 행 × 월 열 테이블
- S-curve (계획선) 자동 생성

### 3단계 — 실적 입력 및 기성 관리
- 월별 실적 공정률 입력
- 계획 vs 실적 S-curve 비교
- 기성청구액 자동 계산
