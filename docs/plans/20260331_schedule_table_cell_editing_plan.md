# 마스터 테이블 셀 편집 강화 계획

작성일: 2026-03-31
기준 저장소: `/home/pengejeen/p6ix-CPE`

## 1. 목표/배경

- `ScheduleMasterList`의 기존 화면 구조를 유지하면서 셀 편집 생산성을 높인다.
- 스프레드시트 엔진 도입 없이 현재 커스텀 테이블 위에 `배치 편집`, `기본 붙여넣기`, `선택 행 일괄 적용`을 추가한다.
- 이 작업은 엑셀급 함수/시트/정렬/필터가 아니라, 현업 입력 속도를 높이는 `셀 편집 UX` 개선에 한정한다.

## 2. 범위(In) / 범위(Out)

- In
  - 활성 셀 추적 및 셀 편집 상태 명확화
  - 선택 행 대상 동일 필드 일괄 적용
  - 단일 컬럼 기준 연속 붙여넣기
  - 붙여넣기/일괄 적용 시 단일 Undo 스텝 보장
  - 숫자/퍼센트/체크박스 기본 검증
- Out
  - 스프레드시트 엔진 도입
  - 함수, 수식 계산, 자동완성 핸들, 직사각형 범위 선택
  - 다중 컬럼 블록 붙여넣기
  - 테이블 레이아웃/카테고리 구조 전면 개편

## 3. 작업 단위

1. 편집 메타 설계
- 편집 가능 필드, 필드 타입, 붙여넣기 허용 범위를 유틸로 정리한다.

2. 스토어 배치 수정 액션 추가
- 여러 셀 변경을 한 번에 적용하는 액션을 추가하고, 재계산과 Undo 경계를 통합한다.

3. 활성 셀/선택 행 편집 연결
- 현재 선택 행과 활성 셀을 결합해 `Ctrl/Cmd+Enter` 일괄 적용을 지원한다.

4. 단일 컬럼 붙여넣기 연결
- 현재 셀에서 아래 방향으로 연속 붙여넣기를 지원한다.
- 선택 행이 여러 개인 경우 단일 값 붙여넣기는 선택 행 전체에 적용한다.

5. 사용자 피드백 정리
- 지원하지 않는 붙여넣기 형식과 검증 실패 건수를 toast로 요약한다.

## 4. 검증 방법

- 수동 검증
  - 동일 필드 다중 행 선택 후 `Ctrl/Cmd+Enter` 적용
  - Excel/Sheets에서 한 컬럼 복사 후 연속 붙여넣기
  - 퍼센트/숫자/체크박스 필드의 유효성 검증
  - 붙여넣기 후 Undo 1회로 전체 되돌리기
- 자동 검증
  - `cd frontend && npm run build`

## 5. 리스크/대응

- `rowSpan` 병합 셀과 일반 셀의 포커스 규칙 충돌
  - 활성 셀은 실제 렌더링된 입력 요소만 대상으로 추적한다.
- 다량 셀 수정 시 계산 로직 중복 실행
  - 배치 액션 내부에서 아이템별 순차 적용으로 통합한다.
- 붙여넣기 형식 다양성
  - v1은 단일 컬럼 위주로 제한하고, 다중 컬럼은 명시적으로 거절한다.

## 6. 롤백 계획

- 신규 유틸과 배치 액션을 제거하고, 기존 `updateItem`/`updateItemsField` 기반 편집 흐름으로 복귀한다.
- 셀 편집 이벤트 연결만 해제하면 기존 저장/간트/AI/스냅샷 흐름은 유지된다.

## 7. 완료 기준(Definition of Done)

- 선택 행 동일 필드 일괄 적용이 동작한다.
- 단일 컬럼 붙여넣기가 동작한다.
- 붙여넣기/일괄 적용이 Undo 1단계로 기록된다.
- 기존 저장, 삭제, 드래그 정렬, 간트 전환 동작이 유지된다.
- 프론트엔드 빌드가 통과한다.
