# 공기검토플랫폼 실행 계획 (30-60-90)

작성일: 2026-03-20  
기준 저장소: `/home/pengejeen/p6ix-CPE`

## 1. 목표

- 기능 확장보다 `핵심 도메인 안정화`, `회귀 테스트 체계 확보`, `배포/운영 기준 정착`을 우선한다.
- 우선순위는 `P0(정확성/보안)` → `P1(운영성/연동 안정화)` → `P2(보조 UX/문서 고도화)` 순서로 적용한다.

## 2. 실행 원칙

- 테스트가 없는 기능 수정은 지양하고, 버그 수정 시 재현 테스트를 반드시 추가한다.
- `TOTAL` 스케줄-가동률-생산성 축은 순차 검증(BE 수정 → QA 검증 → FE 연동)으로 진행한다.
- 외부 연동(Keycloak/Google/Gemini/공공 API)은 mock 기반 자동화 테스트와 수동 체크리스트를 병행한다.

## 3. 2~4주 즉시 실행 백로그

| 구분 | 작업 | 우선순위 | 난이도 | 담당 |
| --- | --- | --- | --- | --- |
| 테스트 | 앱별 스모크 테스트 구축(`user`, `operatio`, `cpe_module`, `cpe_all_module`) | P0 | M | QA, BE |
| 보안 | `cpe_all_module` 주요 ViewSet 인증 강제 및 회귀 테스트 | P0 | M | BE |
| 백엔드 | 프로젝트/가동률/스케줄 저장 흐름 회귀 테스트 1차 세트 | P0 | L | BE, QA |
| 프론트엔드 | 저장-재조회-오류 표시 UX 점검(`ScheduleMasterList`, `TotalCalc`, `OperatingRate`) | P0 | L | FE |
| 운영/문서 | 배포 검증 순서 문서화(`check -> 영향 앱 테스트 -> 전체 테스트 -> frontend build -> docker build`) | P0 | S | DevOps, PM |
| 데이터 | 기상/공휴일 적재 명령 검증 및 재실행 절차 문서화 | P1 | M | Data, BE |
| 인증 | SSO/세션/legacy JWT 경계 테스트 추가 | P1 | M | BE, QA |
| 문서 | API 핵심 엔드포인트 명세화 및 변경이력 기준 정착 | P1 | M | Documentation, PM |

## 4. 30-60-90일 로드맵

## 4.1 30일 (기준선 확보)

- 목표:
  - 기능 상태 매트릭스(구현/불완전/미구현) 확정
  - 치명 경로 스모크 테스트 정착
- 핵심 작업:
  - 핵심 API(프로젝트, 로그인/세션, 가동률, 스케줄 저장/초기화, 내보내기) 최소 검증
  - 로컬/CI 실행 환경 표준화
- 산출물:
  - 기능 상태 매트릭스
  - 스모크 테스트 세트 v1
  - 로컬 실행/검증 가이드
- 완료 기준(DoD):
  - `manage.py check`, `manage.py test`, `npm run build`, Docker build 재현 가능
  - 상위 사용자 흐름 5개 이상 테스트/체크리스트 확보

## 4.2 60일 (핵심 회귀 방지)

- 목표:
  - `TOTAL` 및 `APARTMENT` 핵심 도메인 자동화 테스트 강화
  - 인증/연동 실패 경로 안정화
- 핵심 작업:
  - 가동률/생산성/스케줄 계산 및 저장 로직 통합 테스트
  - 프로젝트 soft delete/소유권/권한 검증
  - SSO/Google 로그인 mock 테스트
- 산출물:
  - 회귀 테스트 세트 v2
  - API 계약서 초안
  - 장애 대응 Runbook 초안
- 완료 기준(DoD):
  - 핵심 시나리오 10개 이상 자동화
  - 알려진 치명 버그가 재현 테스트로 고정

## 4.3 90일 (운영 체계 정착)

- 목표:
  - 릴리즈/롤백/장애 대응 절차를 문서와 실행으로 검증
  - 품질 기준과 기술부채 백로그 운영
- 핵심 작업:
  - 배포 리허설
  - 성능 병목 점검(대형 스케줄 저장, 가동률 재계산, 보고서 생성)
  - 문서 정합성(SRS/설계서/사용자 가이드/명세서) 정리
- 산출물:
  - 릴리즈 운영 가이드
  - `CHANGELOG.md` 기준
  - 기술부채 백로그
- 완료 기준(DoD):
  - 운영 절차 인수인계 가능한 수준으로 표준화
  - 문서 간 충돌 항목 정리 완료
