초록(Abstract)

  본 프로젝트(건설 공기산정 시스템)는 건설 프로젝트의 공사기간을 표준 생산성 데이터와 현장 조건, 가동률, 기후/공휴일 정보를 반영해 산정·관리하는 웹 기반 시스템이다. 사용자는 프로젝트를 생성해 공사 개요와 공법·근무 조건을 입력하고, 기상·휴일 데이터를 포함한 기준으로 공기 산정과 공종별 기간 요약/견적을 얻을 수 있다. 시스템은 Django REST 백엔드와 React 프론트엔드로 구성되며, 생산성 기준 데이터(CIP/말뚝 등)와 공정 스케줄 정보를 관리하고 일부 AI 요약 기능을 제공한다.

# **소개(Introduction)**

## **1.1.**  **목적**

##   본 문서는 P6IX 건설 공기산정 시스템의 요구사항을 명확히 정의하여 이해관계자 간 합의를 돕고, 개발·검증 범위를 일관되게 관리하기 위한 기준으로 사용한다. 시스템은 공사기간 산정에 필요한 기준 데이터, 현장 조건, 가동률 및 기상·공휴일 정보를 통합해 산정 결과와 근거를 제공하는 것을 목적으로 한다.

## **1.2.**  **범위**

  P6IX는 웹 기반 공기산정 플랫폼으로, 프로젝트 생성 및 관리, 공사 개요/조건 입력, 공종별 공기 산정, 가동률 계산, 생산성 기준 데이터 관리, 공정 스케줄(JSON) 저장, 공사기간 견적서 및 요약 제공을 포함한다. 기상 일자료와 공휴일 데이터를 수집·저장하여 산정에 활용하며, 일부 AI 기반 요약 기능을 지원한다. 본 범위는 공정 실행 제어, 현장 자동화, 외부 ERP/PMIS 연동과 같은 운영·제어 시스템 기능은 포함하지 않는다.

## **1.3.**  **용어 및 정의**

* **공기산정**: 프로젝트의 공사기간을 기준 데이터와 조건을 반영해 계산하는 과정.  
* **가동률**: 기후·휴일 조건을 고려한 연간 작업 가능 비율(%) 또는 작업 가능일수.  
* **표준품셈**: 공종별 표준 생산성/작업량 기준 데이터.  
* **CIP**: 흙막이 공법(지중연속벽 공법) 중 CIP 관련 생산성 산정 기준.  
* **기성말뚝**: 공장에서 제작된 말뚝을 현장에 타설하는 공법 및 그 생산성 기준.  
* **현장타설말뚝(현장타설)**: 현장에서 말뚝을 제작·타설하는 공법 및 그 생산성 기준.  
* **공정 스케줄**: 프로젝트 공사 세부 항목을 JSON 형태로 저장한 일정 데이터.  
* **프로젝트**: 공기산정의 단위가 되는 공사 계획 및 관련 입력/산정 결과의 집합.

| **가동률** | 가동률 기준 불러오기: 프로젝트별 가동률 기준정보(대공종 및 필요 시 구분 단위)를 조회한다.<br>대공종/구분 단위 관리: 스케줄의 `main_category`(대공종)와 `process`(구분)를 기준으로 가동률 산정 대상을 구성하며, 대공종+구분 결합 키(구분 커스텀)도 지원한다. 구분 가동률은 기본적으로 대공종 값을 상속(INHERIT)하고 필요 시 개별(CUSTOM)로 분리 설정한다.<br>지역/기간/근무조건 설정: 지역(기상관측소), 데이터 기간(최근 N년), 공공/민간 구분, 주간 작업일수(5/6/7일)를 설정한다(프로젝트 전역 설정 + 대공종/구분별 설정 지원).<br>기후 불능일 기준 설정: 동절기(최저/평균/최고 선택 + 기준온도 + 적용여부), 혹서기(기준온도), 강우량, 강설량, 순간최대풍속, 비시정, 미세먼지 기준을 설정/활성화한다.<br>산출 결과: 연간 작업일, 기후불능일(법정공휴일과 중복 시 중복 제외), 법정공휴일 수를 산출하고 최종 가동률(%)을 계산한다.<br>저장/반영: 설정/기준값을 저장하면 산출 결과가 갱신되며, 공사기간 산정(스케줄) 계산 시 해당 가동률(%)을 우선 적용한다.<br>자동 생성: 스케줄 저장 시 새 대공종이 추가되면, 해당 대공종의 가동률 기준정보가 없을 경우 기본값으로 자동 생성 후 산출한다. |

| **CIP 생산성 근거** |  직경 규격 선택 CIP 천공 직경(규격)을 선택·저장한다. 지층별 깊이 입력 점질토/사질토/풍화암/연암/경암/혼합층 깊이(m)를 입력·저장한다. 비트 타입 선택 지층별 비트 타입(오거비트/개량형비트/해머비트)을 선택·저장한다. 준비시간·철근망/H형강 입력 준비시간(t1)과 철근망·H형강 건입 시간(t3)을 입력한다. 콘크리트 타설 입력 콘크리트 타설 길이(m)를 입력하여 타설 시간 산정에 반영한다. 작업계수 입력 작업계수(f)를 입력·저장한다. 굴착·타설 시간 계산 총 굴착 깊이, 천공시간(t2), 콘크리트 타설시간(t4)을 계산한다. 소요시간/일일 생산성 산출 본당 소요시간(T)과 일일 작업량(공)을 산출·저장한다. 계산식 저장 산정에 사용된 계산식을 저장해 근거를 제공한다. 천공 기준표 관리 비트 타입·직경 규격별 천공 기준을 관리한다. |
| **기성말뚝 생산성 근거** | 말뚝 직경/규격 선택 말뚝 직경 규격을 선택·저장한다. 지층별 깊이 입력 점질토/사질토/풍화암/연암/경암/혼합층 깊이(m)를 입력·저장한다. 말뚝 종류 결정 지층별 말뚝 종류(오거비트/개량형비트/해머비트 등)를 선택·저장한다. 작업시간 입력 준비시간(t1), 근입·항타 시간(t3), 용접 시간(t5), 그라우팅 길이 등을 입력한다. 작업계수 입력 작업계수(f)를 입력·저장한다. 계산 결과 산출 총 깊이, 항타시간(t2), 그라우팅시간(t4), 사이클타임, 일일 생산성(본)을 계산·저장한다. 기준표 관리 말뚝 종류·직경 규격별 기준값(분/m)을 관리한다. |

# **4\.**     **비 기능 요구사항 (Nonfunctional Requirements)**

## **4.1.**  **시스템 기본요건**

### **4.1.1.**  **성능**

일반 인터넷 수준의 응답 수준을 요구한다.

보급형 컴퓨터에서 무난하게 수행이 가능하다.

### **4.1.2.**  **신뢰성**

서버가 장애 없이 돌아가도록 한다.

지속성에 대한 척도로 장애가 발생하기까지의 시간, 즉 MTTF(Mean Time To Failure)로 표현

### **4.1.3.**  **편의성**

인터넷이 가능한 어느 곳에서든 사용자 인증 후에 서비스를 이용할 수 있다.

### **4.1.4.**  **유지보수성**

프로젝트를 세분화 모듈화 하여 모든 부분을 부품화하여 수정, 추가할 때 용이하게 설계

### **4.1.5.**  **보안 및 인증체계**

템플릿 생성시 여타의 웹 사이트의 보안 인증체계를 따른다.

### **4.1.6.**  **가용성**

서버가 작동하고 있는 한 어떤 시간이든 시스템을 활용할 수 있으며 또한 만들어져 있는 템플릿 또한 언제든지 열람이 가능하다.

### **4.1.7.**  **복구성**

시스템 장애 즉, 웹 서버나 데이터베이스 서버가 다운되었을 때 대체 서버를 이용하여 항상 대기 시켜 놓으면서 DB 동기화를 시켜 놓는다.
